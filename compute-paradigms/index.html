// STATEFUL: WebSocket Game Server

# Connection → Server A

WS /game/room/abc

✗ Server dies → session lost | ✗ Can't migrate | ✗ Sticky sessions

$ diff stateless stateful

Stateless Nothing in memory → DB/Redis = truth

Stateful Data in memory → Memory = truth

Stateless Crash? No problem, spin new instance

Stateful Crash? Data gone, user angry

Pro tip: Externalize state to Redis/ElastiCache

$ sudo rm -rf /state/*

Stateless apps are like goldfish:

[1] No memory of previous requests

[2] Store state elsewhere (DB/Redis/S3)

[3] Any instance handles any request

[4] kill -9 without drama
